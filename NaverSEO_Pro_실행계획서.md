## 📋 시작 전 준비 (Day 0)

### 1. 필수 계정 생성 & API 키 발급

| 서비스 | 용도 | 발급 URL | 비용 |
|--------|------|----------|------|
| **Supabase** | DB + 인증 | https://supabase.com | 무료 (50K MAU) |
| **Vercel** | 배포 | https://vercel.com | 무료 (Hobby) |
| **Anthropic** | AI 콘텐츠 생성 | https://console.anthropic.com | $5 크레딧으로 시작 |
| **네이버 검색광고** | 키워드 검색량 API | https://searchad.naver.com | 무료 |
| **네이버 개발자센터** | 검색/데이터랩 API | https://developers.naver.com | 무료 |
| **토스페이먼츠** | 한국 결제 | https://developers.tosspayments.com | 거래 수수료만 |
| **도메인** | naverseo.pro 등 | Namecheap 또는 가비아 | ~$10/년 |

### 2. 네이버 API 키 발급 상세 가이드

**검색광고 API (키워드 도구):**
1. https://searchad.naver.com 접속 → 로그인
2. 우측 상단 "광고시스템" 클릭
3. 도구 → API 사용관리
4. "네이버 검색광고 API 서비스 신청" 클릭
5. 발급되는 3가지 저장: `내 계정(CUSTOMER_ID)`, `엑세스라이선스(API_KEY)`, `비밀키(SECRET_KEY)`

**네이버 개발자센터:**
1. https://developers.naver.com 접속 → 로그인
2. Application → 애플리케이션 등록
3. 사용 API 선택: "검색", "데이터랩(검색어트렌드)"
4. 발급되는 2가지 저장: `Client ID`, `Client Secret`

### 3. 개발 환경 세팅

```bash
# Node.js 20+ 설치 확인
node --version

# Claude Code 설치 (이미 있다면 스킵)
npm install -g @anthropic-ai/claude-code

# 프로젝트 디렉토리 생성
mkdir naverseo-pro && cd naverseo-pro

# CLAUDE.md 파일을 이 디렉토리에 복사
# (함께 제공된 CLAUDE.md 파일)

# Claude Code 시작!
claude
```

## 🗓️ Day 6~7: 핵심 기능 #1 — 키워드 리서치

### Claude Code에 입력할 프롬프트

```
네이버 키워드 리서치 기능을 만들어줘.

기능:
1. 키워드 입력 → 네이버 검색광고 API로 검색량 조회
2. 결과 테이블: 키워드, PC 검색량, 모바일 검색량, 경쟁도, 추천점수
3. 추천점수 = (검색량 합계) / (경쟁도 * 블로그 발행문서수) — 높을수록 좋은 키워드
4. 연관 키워드 자동 추출 (API에서 제공)
5. 네이버 데이터랩 API로 최근 12개월 트렌드 차트 표시
6. 결과 CSV 다운로드 기능
7. Supabase에 조회 결과 저장 (나중에 다시 볼 수 있게)

API 구현:
- /api/naver/keywords — 검색광고 API 프록시 (HMAC-SHA256 서명 필요)
- /api/naver/trend — 데이터랩 API 프록시
- /api/naver/blog-count — 특정 키워드의 블로그 발행 문서수 조회

CLAUDE.md의 네이버 API 엔드포인트 정보 참고.
네이버 검색광고 API는 HMAC-SHA256 서명이 필요해.
서명 생성법: timestamp + method + uri를 SECRET_KEY로 HMAC-SHA256 해시.
```

## 🗓️ Day 10~11: 핵심 기능 #3 — SEO 점수 체커

### Claude Code에 입력할 프롬프트

```
네이버 블로그 SEO 점수 체커를 만들어줘.

기능:
1. 입력: 타겟 키워드 + 블로그 글 내용 (텍스트 붙여넣기 또는 저장된 콘텐츠)
2. 분석 항목 (각 항목 0~100점, 가중 평균으로 총점 계산):
   
   a. 제목 최적화 (20%):
      - 키워드 포함 여부
      - 제목 길이 (30~50자 최적)
      - 키워드 위치 (앞쪽일수록 좋음)
   
   b. 본문 최적화 (30%):
      - 키워드 밀도 (1~3% 최적)
      - 관련 키워드 포함 여부
      - 본문 길이 (2000자 이상 권장)
      - 소제목(H2, H3) 사용 여부
   
   c. 구조 점수 (20%):
      - 소제목 개수 (3개 이상 권장)
      - 문단 길이 (너무 긴 문단 경고)
      - 이미지 태그 포함 여부
      - 목록/표 활용 여부
   
   d. 가독성 점수 (15%):
      - 평균 문장 길이
      - 전문 용어 비율
      - 접속사 활용
   
   e. 기술적 SEO (15%):
      - 태그 개수 (5~10개 권장)
      - 메타 디스크립션 길이

3. 결과 UI:
   - 총점 원형 게이지 (점수에 따라 색상: 빨강/노랑/초록)
   - 항목별 점수 바 차트
   - 구체적 개선 제안 목록 (AI가 생성)
   - "AI로 개선하기" 버튼 → 콘텐츠 생성 페이지로 이동

이 분석은 프론트엔드에서 처리해도 되고, 
개선 제안은 Claude API를 짧게 호출해서 생성해.
```

## ⚠️ 주의사항

### 절대 하지 말 것
1. **인증/결제 코드를 AI가 처음부터 작성하게 하지 마세요** → Supabase Auth, 토스페이먼츠 공식 SDK 사용
2. **API 키를 코드에 하드코딩하지 마세요** → 반드시 .env.local 사용
3. **네이버 API를 클라이언트에서 직접 호출하지 마세요** → API Route를 프록시로 사용 (키 노출 방지)
4. **처음부터 완벽한 제품을 만들려 하지 마세요** → MVP 6개 기능에만 집중
